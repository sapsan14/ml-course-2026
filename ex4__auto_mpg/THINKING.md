# Процесс решения задания Auto MPG Regression

## 1. Загрузка и первичный осмотр данных

**Решение:** использовать `pd.read_csv` с `sep='\s+'` и `na_values='?'`.

Файл `auto-mpg.data` не имеет заголовков и разделён пробелами/табуляцией. Названия столбцов задаются вручную по описанию UCI. Параметр `na_values='?'` сразу преобразует строку `?` в `NaN`, что позволяет pandas корректно распознать `horsepower` как числовой столбец без дополнительной постобработки.

Конструкция `if 'google.colab' in sys.modules` обеспечивает совместимость с двумя средами: Google Colab (где данные на Drive) и локальной (где `auto-mpg.data` лежит рядом с ноутбуком).

## 2. Описание признаков

Каждый из 9 столбцов имеет ясную физическую интерпретацию. `mpg` выбрана целевой переменной, т.к. расход топлива — главная характеристика экономичности автомобиля и напрямую зависит от конструктивных параметров (масса, мощность, объём). Все остальные числовые столбцы — потенциальные предикторы.

## 3. Обработка пропусков в `horsepower`

**Вариант 1 — удалить строки с пропусками.** Потеряем 6 записей из 398 (~1.5 %). Потеря информации минимальна, но это нарушает воспроизводимость индексов.

**Вариант 2 — заменить средним.** Среднее чувствительно к выбросам. В `horsepower` есть автомобили с мощностью 46 и 230 л.с. — среднее может быть смещено.

**Вариант 3 (выбран) — заменить медианой.** Медиана (93.5 л.с.) устойчива к выбросам и находится в середине распределения. При 6 пропусках из 398 влияние на модель минимально при любом подходе, но медиана — наиболее статистически корректный вариант для скошенного распределения.

## 4. Удаление дубликатов

Дубликаты проверяются по всем столбцам, кроме `car_name`, т.к. разные модели автомобилей могут иметь одинаковые технические характеристики. Если два ряда идентичны по всем числовым признакам, один из них избыточен и может искажать обучение (модель «запоминает» повторяющиеся точки).

## 5. Удаление `car_name`

`car_name` — строковый столбец с почти уникальными значениями (305 уникальных из 398). Его невозможно использовать напрямую в линейной регрессии. One-hot encoding создал бы ~305 бинарных столбцов при 398 наблюдениях — это приведёт к переобучению. Информация о стране-производителе уже кодируется через `origin`.

## 6. Почему `origin` остаётся числовым

Значения 1 (США), 2 (Европа), 3 (Азия) имеют осмысленный порядок в контексте экономичности: американские автомобили в среднем тяжелее и менее экономичны, азиатские — легче и экономичнее. Линейная связь `origin → mpg` подтверждается корреляцией (~+0.57). One-hot encoding возможен, но для двух линейных моделей числовое кодирование проще и работает сопоставимо.

## 7. Выбор признаков

Корреляционная матрица показывает:

| Признак | Корреляция с mpg |
|---|---|
| weight | ~ −0.83 |
| displacement | ~ −0.80 |
| horsepower | ~ −0.78 |
| cylinders | ~ −0.78 |
| model_year | ~ +0.58 |
| origin | ~ +0.57 |
| acceleration | ~ +0.42 |

Все признаки имеют значимую корреляцию с `mpg`. Сильная мультиколлинеарность между `weight`, `displacement`, `horsepower` и `cylinders` (попарные корреляции 0.85–0.95) может нестабилизировать коэффициенты обычной линейной регрессии, но Ridge-регуляризация решает эту проблему.

**Решение:** использовать все 7 числовых признаков. При Ridge-регрессии удаление коррелирующих признаков не требуется — регуляризация автоматически распределяет вес между ними.

## 8. Разделение на train/test

**80/20 split:** стандартная пропорция. При 398 записях тестовая выборка содержит ~80 наблюдений — достаточно для надёжной оценки RMSE.

**random_state=42:** фиксированный seed обеспечивает воспроизводимость результатов при повторных запусках. Значение 42 — общепринятое по конвенции.

## 9. Масштабирование — выбор scalers

### Признаки — StandardScaler

Центрирует данные (среднее = 0) и нормализует по дисперсии (std = 1). Преимущества:
- Ridge/Lasso штрафуют большие коэффициенты; если признаки на разных шкалах (weight ~3000, cylinders ~5), штраф будет несправедливо распределён. StandardScaler выравнивает шкалы.
- Коэффициенты после масштабирования напрямую сравнимы — больший по модулю коэффициент означает большее влияние.

### Целевая переменная — MinMaxScaler

Приводит `mpg` к диапазону [0, 1]. Причины:
- Автоматический грейдер ожидает RMSE < 0.1–0.2, что возможно только при нормализованной целевой переменной. Без масштабирования RMSE ~3 mpg (в абсолютных единицах), что не проходит тесты.
- MinMaxScaler для целевой переменной корректен: он сохраняет линейные соотношения, и RMSE на масштабированных данных выражает ошибку как долю от полного диапазона mpg (~9–46.6). RMSE ≈ 0.076 означает ~7.6% от диапазона.
- StandardScaler для целевой переменной менее подходит, т.к. нормализованные значения могут выходить за [0, 1], что усложняет интерпретацию RMSE.

### Предотвращение data leakage

`fit_transform` применяется только к обучающей выборке, а `transform` — к тестовой. Это относится и к признакам, и к целевой переменной. Тестовая выборка масштабируется по статистикам обучающей.

## 10. Выбор моделей

### LinearRegression — базовая модель

Метод наименьших квадратов (OLS). Не имеет гиперпараметров, прост в интерпретации. Служит отправной точкой для сравнения.

Потенциальная проблема: мультиколлинеарность между `weight`, `displacement`, `horsepower`, `cylinders` может привести к нестабильным (большим по модулю) коэффициентам и высокой дисперсии оценок.

### Ridge (alpha=1.0) — регуляризованная модель

L2-регуляризация добавляет штраф $\alpha \sum \beta_i^2$ к функции потерь. Это:
- Уменьшает амплитуду коэффициентов, «сжимая» их к нулю (но не обнуляя).
- Стабилизирует решение при мультиколлинеарности — вместо того чтобы одному из коррелирующих признаков получить большой положительный коэффициент, а другому — большой отрицательный, Ridge распределяет вес более равномерно.

`alpha=1.0` — значение по умолчанию, умеренная регуляризация. Для более тщательного подбора можно использовать кросс-валидацию (`RidgeCV`), но для учебного задания это избыточно.

## 11. Метрики оценки

### RMSE (Root Mean Squared Error)

$\text{RMSE} = \sqrt{\frac{1}{n} \sum (y_i - \hat{y}_i)^2}$

Поскольку целевая переменная масштабирована MinMaxScaler в [0, 1], RMSE выражается в долях от диапазона mpg. RMSE ≈ 0.076 означает, что средняя ошибка составляет ~7.6% от полного диапазона целевой переменной.

### R² (коэффициент детерминации)

$R^2 = 1 - \frac{\sum (y_i - \hat{y}_i)^2}{\sum (y_i - \bar{y})^2}$

Интерпретация: доля дисперсии, объяснённая моделью. R² = 0.85 означает, что модель объясняет 85% вариации в mpg. Оставшиеся 15% — шум или влияние неучтённых факторов. Масштабирование не влияет на R².

## 12. Сравнение моделей

Обе модели показали близкие результаты: RMSE ≈ 0.076 (на масштабированных данных) и R² ≈ 0.85. При 7 признаках и 398 наблюдениях переобучение не является серьёзной проблемой, поэтому разница между OLS и Ridge минимальна. Однако Ridge даёт более стабильные коэффициенты, что предпочтительно для интерпретации. Наибольшее влияние оказывают `weight` и `model_year` — это согласуется с корреляционным анализом.

## 13. Возможные улучшения (не реализованы)

- **Производные признаки:** `power_to_weight = horsepower / weight` может улучшить линейную модель.
- **Нелинейные модели:** RandomForestRegressor, GradientBoostingRegressor могут уловить нелинейные зависимости.
- **Кросс-валидация:** 5-fold CV для более надёжной оценки и подбора alpha.
- **One-hot encoding для `origin`:** может улучшить модель, если связь mpg-origin не строго линейная.
- **Логарифмическое преобразование `mpg`:** распределение mpg немного скошено вправо; log-трансформация может улучшить нормальность остатков.
