## Мои размышления и анализ

### Что я знаю о датасете Breast Cancer (Wisconsin)

Классический датасет sklearn. Основные характеристики:
- **569 образцов**, 30 числовых признаков
- Два класса: **malignant (0)** — злокачественный (212 образцов), **benign (1)** — доброкачественный (357 образцов)
- Признаки вычислены из оцифрованного изображения клеточного ядра (FNA)
- 10 базовых измерений × 3 варианта (mean, standard error, worst): radius, texture, perimeter, area, smoothness, compactness, concavity, concave points, symmetry, fractal dimension
- **Нет пропущенных значений** — данные чистые
- Все признаки числовые (float64) — не нужно кодирование категорий

### Ключевые особенности KNN

1. **KNN — «ленивый» алгоритм:** не строит модель при обучении, а хранит все обучающие точки и делает предсказание в момент запроса.
2. **Евклидово расстояние** чувствительно к масштабу признаков. Если один признак имеет диапазон [0, 1000], а другой — [0, 1], первый будет доминировать при вычислении расстояния. **Поэтому StandardScaler обязателен.**
3. **Значение k:**
   - Малое k (например, 1–3) → высокая дисперсия (variance), низкое смещение (bias) → склонность к переобучению
   - Большое k (например, 15–25) → низкая дисперсия, высокое смещение → склонность к недообучению
   - Обычно хороший выбор: k = 5–7 для этого датасета
4. **Recall для malignant (label=0):** Это критически важная метрика — лучше ошибочно диагностировать здорового как больного (ложноположительный), чем пропустить злокачественную опухоль (ложноотрицательный).

### Реализация KNN вручную

Алгоритм `my_knn_predict_single(X_train, y_train, x_new, k=5)`:
1. Вычислить евклидово расстояние от `x_new` до каждой точки в `X_train`:
   - `distances = np.sqrt(np.sum((X_train - x_new) ** 2, axis=1))` 
   - Или `np.linalg.norm(X_train - x_new, axis=1)`
2. Найти индексы k ближайших точек: `np.argsort(distances)[:k]`
3. Получить их метки классов: `y_train[nearest_indices]`
4. Мажоритарное голосование: `Counter(nearest_labels).most_common(1)[0][0]`

`my_knn_predict(X_train, y_train, X_new, k=5)`:
- Просто вызвать `my_knn_predict_single` для каждой строки `X_new`
- Вернуть массив предсказаний

### Важное замечание: масштабированные данные

В шаблоне функции KNN принимают `X_train` и `X_test` как аргументы. Но **предсказывать нужно на масштабированных данных** (`X_train_scaled`, `X_test_scaled`), иначе результат будет плохим. При вызове:
```python
my_preds = my_knn_predict(X_train_scaled, y_train, X_test_scaled, k=5)
```

## Потенциальные ловушки

1. **`pos_label=0` в recall_score** — по умолчанию sklearn считает recall для класса 1; нужно явно указать `pos_label=0` для malignant
2. **fit_transform vs transform** — scaler.fit_transform только на train, scaler.transform на test
3. **Тип возвращаемого значения** — `my_knn_predict_single` должен возвращать int (метку класса), а `my_knn_predict` — массив/список
4. **Масштабированные данные в KNN** — не перепутать: передавать `X_train_scaled`/`X_test_scaled`, а не `X_train`/`X_test`
5. **Counter работает с массивами** — если `y_train[nearest]` — numpy array, Counter может работать корректно, но лучше явно преобразовать метки в list: `Counter(y_train[nearest].tolist())`
6. **Имя столбца в DataFrame** — в шаблоне написано "klass" (строка 13 шаблона), но в комментарии сказано "diagnosis" (строка 65) — нужно проверить, какое имя ожидает тестер. В шаблоне: `# Loo DataFrame tunnustest ja lisa veerg "diagnosis"` → столбец называется `diagnosis`

## Мои размышления и анализ

### Что я знаю о датасете Titanic

Классический датасет Kaggle. Столбцы обычно включают:
- `PassengerId` — ID пассажира (бесполезен для модели)
- `Survived` — выжил или нет (0/1) — **целевая переменная**
- `Pclass` — класс каюты (1/2/3) — сильный предиктор
- `Name` — имя (можно извлечь титул: Mr, Mrs, Miss и т.д.)
- `Sex` — пол — **один из сильнейших предикторов** ("женщины и дети первыми")
- `Age` — возраст — есть пропуски (~20%)
- `SibSp` — кол-во братьев/сестёр/супругов на борту
- `Parch` — кол-во родителей/детей на борту
- `Ticket` — номер билета (сложно использовать)
- `Fare` — стоимость билета — коррелирует с Pclass
- `Cabin` — номер каюты — очень много пропусков (~77%)
- `Embarked` — порт посадки (C/Q/S) — есть пару пропусков

### Стратегия предобработки

1. **Пропуски:**
   - `Age`: заполнить медианой (или медианой по группам Pclass/Sex)
   - `Cabin`: слишком много пропусков → удалить столбец (или извлечь палубу из первой буквы)
   - `Embarked`: заполнить модой (наиболее частое значение — 'S')
   
2. **Кодирование категорий:**
   - `Sex`: бинарное кодирование (male=0, female=1)
   - `Embarked`: one-hot encoding или label encoding
   
3. **Удаление бесполезных столбцов:**
   - `PassengerId`, `Name`, `Ticket` — не несут прямой предсказательной силы
   - (Из `Name` можно извлечь титул как feature engineering)

4. **Выбросы:**
   - `Fare` может иметь экстремальные значения → проверить boxplot
   - `Age` — проверить на наличие нереалистичных значений

### Стратегия создания двух моделей

Задание требует **как минимум две модели логистической регрессии с разными признаками**. Моя стратегия:

**Модель 1 — Базовая (минимум признаков):**
- Использовать только самые сильные предикторы: `Pclass`, `Sex`, `Age`
- Простая, интерпретируемая модель

**Модель 2 — Расширенная (больше признаков + feature engineering):**
- `Pclass`, `Sex`, `Age`, `SibSp`, `Parch`, `Fare`, `Embarked`
- Возможно: новый признак `FamilySize = SibSp + Parch + 1`
- Возможно: признак `IsAlone` (один ли пассажир)
- Возможно: `Title` из имени

Ожидание: Модель 2 должна показать лучший F1-score.

### Метрики оценки

Нужно посчитать для обеих моделей:
- **Accuracy** — общая точность
- **Precision** — точность предсказания положительного класса
- **Recall** — полнота (чувствительность)
- **F1-score** — гармоническое среднее precision и recall
- **AUC-ROC** — площадь под ROC-кривой

Использовать `sklearn.metrics`: `accuracy_score`, `precision_score`, `recall_score`, `f1_score`, `roc_auc_score`.

## Мои размышления и анализ

### Что я знаю о датасете Mall Customers

Классический учебный датасет для кластеризации. 200 записей клиентов торгового центра:
- **CustomerID** — идентификатор (не используется)
- **Gender** — пол (Male/Female) — не используется в этом задании
- **Age** — возраст (18–70) — не используется в этом задании
- **Annual Income (k$)** — годовой доход в тысячах долларов (15–137)
- **Spending Score (1-100)** — балл расходов, присвоенный ТЦ на основе поведения (1–100)

Нет пропущенных значений. Для кластеризации используются только 2 признака: доход и балл расходов.

### Что я знаю о K-Means

1. **K-Means** делит данные на k кластеров, минимизируя сумму квадратов расстояний от точек до центроидов (инерцию).
2. **Метод локтя:** строим график инерции по k, ищем «изгиб» — точку, после которой уменьшение инерции замедляется. Для этого датасета типичный optimal_k = 5.
3. **StandardScaler обязателен:** хотя оба признака имеют сопоставимые масштабы (доход 15–137, балл 1–100), стандартизация всё равно улучшает результат, т.к. приводит к нулевому среднему и единичной дисперсии.
4. **random_state=42** обеспечивает воспроизводимость, **n_init=10** — KMeans запустится 10 раз с разными начальными центроидами и выберет лучший результат.

### Ожидаемые 5 кластеров

На этом датасете K-Means с k=5 обычно выделяет следующие сегменты:
1. **Высокий доход, высокие расходы** — «премиум-клиенты» (правый верхний угол)
2. **Высокий доход, низкие расходы** — «экономные состоятельные» (правый нижний угол)
3. **Низкий доход, высокие расходы** — «расточительные» (левый верхний угол)
4. **Низкий доход, низкие расходы** — «экономные» (левый нижний угол)
5. **Средний доход, средние расходы** — «обычные клиенты» (центр)

### Маркетинговые идеи для каждого сегмента

- **Премиум:** VIP-программа, эксклюзивные бренды, персональные предложения
- **Экономные состоятельные:** стимулирование расходов через скидки, акции, кешбэк
- **Расточительные:** программы лояльности с вознаграждениями, кредитные предложения
- **Экономные:** базовые акции, скидки на товары повседневного спроса
- **Обычные:** общие маркетинговые кампании, сезонные распродажи
